# My 페이지 API 연동

## 목표

- My 페이지의 dummy 기반 설정/프로필 흐름을 실제 API 기준으로 전환한다.
- 프로필 수정, 설정 저장, 계정 삭제 흐름을 정리한다.

## 범위

- `src/pages/My.tsx`
- 관련 profile / settings API 호출부

## 현재 상태

- 아바타 업로드/삭제는 이미 연결되어 있다.
- 닉네임 저장, 설정 저장, 계정 삭제, 자동 이월 토글 등은 dummy 데이터 기반이 남아 있다.
- 페이지 내부에 프로필/설정 UI와 업로드 로직이 함께 모여 있다.

## 작업 단계

1. My 페이지에서 실제 API로 대체할 항목을 정리한다.
2. 저장/삭제 액션별 호출 흐름과 후처리를 확정한다.
3. dummy 상태를 제거하고 실제 응답 기준으로 UI를 재구성한다.
4. 필요하면 profile / settings 섹션을 분리한다.
5. 완료 후 auth/settings 문서를 갱신한다.

## 주요 결정 포인트

- 저장 단위를 프로필/설정으로 나눌지
- 계정 삭제 후 라우팅과 세션 정리를 어디서 담당할지

## 리스크

- 아바타 업로드 흐름과 일반 저장 흐름이 뒤섞이면 페이지 책임이 다시 커질 수 있다.
- 삭제 이후 세션 정리와 라우팅이 auth 흐름과 엮인다.

## 완료 조건

- dummy 기반 설정값이 제거된다.
- My 페이지 주요 액션이 실제 API와 연결된다.
- 계정 삭제 이후 흐름이 명확히 정리된다.
