# DailyLog API 연동

## 목표

- DailyLog 페이지를 테스트용 로컬 데이터 기반에서 실제 API 기반으로 전환한다.
- 자동저장, 삭제, 목록 조회, 검색 흐름을 실제 데이터 기준으로 정리한다.

## 범위

- `src/pages/DailyLog.tsx`
- `src/api/generated/daily-logs/daily-logs.ts`

## 현재 상태

- 페이지 내부에 테스트용 로그 배열과 임시 흐름이 남아 있다.
- 생성된 DailyLog API 코드는 존재하지만 실제 편집/조회 흐름과 완전히 연결되어 있지 않다.

## 작업 단계

1. 현재 DailyLog UI 흐름과 API 스펙을 대조한다.
2. 자동저장 시점과 저장 실패 처리 기준을 정한다.
3. 삭제 / 기간별 조회 / 검색 흐름을 실제 API로 연결한다.
4. 테스트용 로그 데이터를 제거한다.
5. 관련 product spec 또는 design doc을 보강한다.

## 주요 결정 포인트

- autosave debounce 기준
- 검색을 전용 API로 처리할지, 목록 조회 기반으로 처리할지

## 리스크

- 입력 중 저장 실패 처리 방식이 불분명하면 사용성 문제가 생길 수 있다.
- 목록/검색 데이터 소스가 분리되면 UI 상태 관리가 복잡해질 수 있다.

## 완료 조건

- DailyLog의 주요 조회/편집 흐름이 실제 API와 연결된다.
- 테스트용 로컬 로그 데이터가 제거된다.
- 저장 및 삭제 실패 처리 기준이 정리된다.
