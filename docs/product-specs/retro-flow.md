# Retro Flow

## 목표

- 날짜별 회고를 카테고리 단위로 작성하고, 선택된 회고와 저장 상태를 명확하게 보여줍니다.

## 핵심 사용자 흐름

1. 사용자는 좌측 목록에서 날짜별 회고 묶음을 선택합니다.
2. 선택한 회고의 카테고리 중 하나를 segmented control로 고릅니다.
3. 카테고리별 질문에 답변을 입력하면 자동 저장되거나 저장 버튼으로 저장합니다.
4. 삭제 시 삭제할 카테고리를 선택하고, 토스트 undo 시간 동안 취소할 수 있습니다.

## 현재 구현 메모

- 생성/수정/삭제 API는 연결되어 있습니다.
- 회고 목록은 아직 테스트용 로컬 데이터 기반입니다.
- 선택된 회고는 카드 강조 상태로 표시됩니다.
- 회고 삭제는 카테고리별 선택 삭제를 지원합니다.

## 예외 및 실패 처리

- 삭제 대상이 없으면 에러 토스트를 보여줍니다.
- 저장 실패 처리와 목록 재동기화는 아직 더 보강이 필요합니다.
- 미래 날짜는 선택할 수 없습니다.

## 후속 작업

- 회고 목록 조회 / 검색 API 연결
- 생성/삭제 후 목록 재조회 또는 캐시 동기화
- 회고 페이지 상태를 controller 훅으로 분리할지 검토
