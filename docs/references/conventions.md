# Conventions

## Documentation

- `README.md`는 프로젝트 소개 문서로 유지합니다.
- 내부 운영 문서는 `docs/`를 source of truth로 사용합니다.
- 긴 단일 문서보다 도메인별 문서로 분리합니다.

## Code Structure

- `components/ui`는 공용 UI만 둡니다.
- 도메인 의미가 있는 컴포넌트는 `features/*/components`에 둡니다.
- 공용 훅은 `src/hooks`, 도메인 훅은 각 feature에 둡니다.
- 공용 `cn` 유틸을 사용하고 로컬 `cx` 중복 선언은 줄입니다.

## Comments

- 흐름 파악에 도움이 되는 설명 주석은 `// INFO:` 형식을 사용합니다.
- 주석은 "무슨 코드인지"보다 "왜 이런 흐름인지", "왜 여기서 필요한지"를 설명합니다.
- 상태 전이, 복구 로직, 전역 실행기, 외부 제약사항 같은 맥락이 있는 코드에 우선 사용합니다.
- 자명한 대입, 단순 렌더링, 이름만 봐도 알 수 있는 코드는 주석을 달지 않습니다.
- 에이전트와 함께 작업할 때도 동일한 규칙을 따릅니다.
