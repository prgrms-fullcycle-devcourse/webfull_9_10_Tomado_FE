# Settings

`settings` 도메인은 앱 전역 설정 성격의 기능 로직을 둡니다.

현재 구현된 범위는 `BGM 플레이어`입니다.  
향후에는 아래 성격의 설정 로직도 이 도메인에 포함될 예정입니다.

- 타이머 기본 시간 조절
- 휴식/장휴식 시간 조절
- 투두 날짜 이동 관련 설정/행동
- 기타 앱 전역 설정

즉, `settings`는 특정 페이지 전용 UI보다 "어디서든 접근 가능한 전역 설정 기능"을 다루는 도메인으로 보면 됩니다.

## 현재 구현 범위

현재 이 도메인에서 동작하는 것은 배경음악 플레이어입니다.

- 트랙 카테고리: `lofi`, `rain`, `cafe`
- 현재 선택 트랙 표시
- 재생 / 일시정지 / 재개
- 이전 / 다음 트랙 이동
- 볼륨 조절
- 화면 전환 후에도 재생 유지
- 새로고침 후 마지막 트랙 / 볼륨 / 재생 위치 복원
- 브라우저 autoplay 정책에 막힐 경우 첫 사용자 인터랙션에서 재생 재시도

## 파일 구조

### `@@@/settings/tracks.ts`

BGM 트랙 메타데이터 정의 파일입니다.

- 카테고리별 제목/설명/카드 이미지 정의
- `src/assets/audio/bgm/*/*` 아래 오디오 파일 자동 수집
- 실제 플레이어가 소비할 `bgmTracks` 생성
- 모달 카드에 쓰는 `bgmPlayerItems` 생성

이 파일은 "무슨 트랙이 있는지"를 정의하는 순수 데이터 레이어입니다.

### `@@@/settings/bgmStorage.ts`

새로고침 복원을 위한 storage 유틸입니다.

- storage key 정의
- 기본 볼륨 정의
- 마지막 트랙 / 볼륨 / 재생 여부 / 재생 위치 초기값 복원

이 파일은 "저장된 값을 어떻게 읽을지"만 담당합니다.

### `@@@/settings/bgmAudioRuntime.ts`

실제 오디오 런타임을 담당하는 파일입니다.

- 전역 `Audio` 싱글턴 생성
- 현재 트랙 동기화
- 현재 시간 동기화
- 이전/다음 트랙 계산
- autoplay 실패 시 재시도 유틸

이 파일은 React 훅이 아니라, 브라우저 오디오를 직접 다루는 실행 레이어입니다.

### `@@@/settings/useBgmPlayer.ts`

UI가 직접 사용하는 BGM 훅입니다.

- zustand store 구독
- 재생 상태 노출
- 현재 트랙 노출
- 볼륨 변경 핸들러
- 재생/일시정지/이전/다음 핸들러
- 카테고리 카드 선택 핸들러

즉 이 훅은 컴포넌트 입장에서 "플레이어를 어떻게 조작할지"만 알면 되도록 감싼 공개 API입니다.

### `@@@/settings/index.ts`

현재 settings 도메인 public export 파일입니다.

외부에서는 이 파일을 통해:

- `useBgmPlayer`
- `bgmTracks`
- `bgmPlayerItems`

를 가져다 쓰면 됩니다.

## 현재 사용 흐름

대략적인 책임 흐름은 아래와 같습니다.

1. `tracks.ts`가 오디오 파일과 메타데이터를 준비
2. `bgmStorage.ts`가 복원할 초기 상태를 읽음
3. `bgmAudioRuntime.ts`가 실제 `Audio` 엘리먼트를 제어
4. `useBgmPlayer.ts`가 store와 runtime을 조합해서 UI용 API를 노출

## 오디오 파일 추가 규칙

오디오 파일은 아래 경로 규칙을 따릅니다.

```txt
src/assets/audio/bgm/
├─ lofi/
├─ rain/
└─ cafe/
```

새 파일을 추가하면 `tracks.ts`의 `import.meta.glob(...)`가 자동으로 수집합니다.

즉:

- 파일 추가
- 폴더 규칙만 맞추기

까지 하면 기본 트랙 목록에는 자동 반영됩니다.

단, 카테고리 메타데이터는 현재 `tracks.ts`의 `categoryMeta`에 정의되어 있으므로,
새 카테고리를 추가할 때는 그쪽도 같이 정의해야 합니다.

## 설계 원칙

- 오디오 파일 자체는 `src/assets/audio/bgm`에 둡니다.
- 설정 도메인의 핵심은 "전역 설정 기능 로직"입니다.
- 훅과 store가 많지 않을 때는 도메인 루트에 둡니다.
- 공용 UI는 `src/components/ui`를 재사용합니다.
- settings 도메인에는 "전역에서 접근 가능하고 페이지를 넘어서 유지되어야 하는 설정 기능"을 둡니다.

## 아직 구현되지 않은 범위

아직 아래 항목은 구조만 염두에 두고 있고, 실제 구현은 하지 않았습니다.

- 타이머 설정값 조절 로직
- 집중/휴식/장휴식 시간 설정값 저장
- 투두 날짜 이동 관련 설정 처리
- settings 전용 store 세분화

즉 현재 README는 BGM 구현을 기준으로 작성되어 있고, settings 도메인은 이후 앱 전역 설정 기능이 더 추가될 수 있도록 열어둔 상태입니다.
